## 📋 Project Overview

This project compares four recommendation system architectures for e-commerce personalization:

1. **Popularity Baseline** — Non-personalized, recommends top popular products
2. **Collaborative Filtering (CF)** — Truncated SVD with mean-centering (50 latent factors)
3. **Content-Based Filtering (CB)** — Cosine similarity over product features
4. **Hybrid Model** — Weighted combination of CF and CB

### Key Results

| Model | RMSE | Precision@10 | Recall@10 | F1@10 | Coverage |
|-------|------|-------------|----------|-------|----------|
| Popularity Baseline | 1.6557 | 0.0085 | 0.0415 | 0.0123 | 2.2% |
| Collaborative Filtering | **1.5748** | 0.0090 | 0.0305 | 0.0122 | **89.0%** |
| Content-Based Filtering | 1.7107 | 0.0075 | 0.0166 | 0.0093 | 46.8% |
| Hybrid Model | **1.5748** | 0.0085 | 0.0233 | 0.0109 | 85.3% |

### Hypothesis Testing Summary

| Hypothesis | Result |
|-----------|--------|
| H1: VIP+Frequent segments account for disproportionate purchases | ✅ Supported |
| H2: CF achieves lower RMSE than CB | ✅ Supported |
| H3: Hybrid outperforms standalone models | ⚠️ Partially Supported |
| H4: Personalized models outperform popularity baseline | ⚠️ Partially Supported |

## 📝 Methodology

1. **Data Generation** — Synthetic e-commerce dataset with latent factor model
2. **EDA** — 25 visualizations covering all aspects of the data
3. **Preprocessing** — Train-test split (80:20), user-item matrix, mean-centering, feature engineering
4. **Model Building** — Four recommendation models with hyperparameter tuning
5. **Evaluation** — RMSE, Precision@10, Recall@10, F1@10, Catalogue Coverage
6. **Hypothesis Testing** — Chi-square test (H1), metric comparisons (H2-H4)
7. **Segment Analysis** — Performance by customer engagement level
